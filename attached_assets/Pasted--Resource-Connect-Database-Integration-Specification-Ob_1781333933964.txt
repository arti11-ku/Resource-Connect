# Resource Connect - Database Integration Specification

## Objective

Integrate a production-ready PostgreSQL database into the existing Resource Connect project.

The database should store all application data permanently and support the AI Resource Allocation Engine.

The integration must preserve the existing project structure, UI design, layouts, navigation, styling, routes, workflows, and user experience.

---

# Important Requirements

## Do Not Modify

* UI design
* Layouts
* Navigation structure
* Existing pages
* Existing content
* Styling and alignment
* Authentication flow
* Dashboard structure

The database should work behind the scenes without changing the visual appearance of the application.

---

# Database Technology

Use:

* PostgreSQL

Requirements:

* Production-ready schema
* Foreign key relationships
* Indexed primary keys
* Environment variable configuration
* Database persistence after deployment

---

# Core Database Tables

## Users

Stores all platform users.

Fields:

* id (Primary Key)
* name
* email
* password_hash
* role
* phone
* created_at

Roles:

* Reporter
* NGO
* Volunteer
* Admin

---

## Reports

Stores disaster and resource requests.

Fields:

* id (Primary Key)
* reporter_id (Foreign Key)
* title
* description
* category
* severity
* location
* status
* created_at

Status:

* Pending
* Under Review
* Approved
* Completed

---

## NGOs

Stores NGO profiles.

Fields:

* id
* user_id
* ngo_name
* registration_number
* contact_details
* address
* created_at

---

## Volunteers

Stores volunteer information.

Fields:

* id
* user_id
* skills
* availability
* current_status
* location

Current Status:

* Available
* Assigned
* Busy

---

## Resources

Stores inventory information.

Fields:

* id
* ngo_id
* resource_name
* category
* quantity
* location
* updated_at

Examples:

* Food Packets
* Water Bottles
* Blankets
* Medical Kits

---

## Tasks

Stores assigned work.

Fields:

* id
* report_id
* ngo_id
* title
* description
* status
* created_at

Status:

* Pending
* Assigned
* In Progress
* Completed

---

## Volunteer Assignments

Stores volunteer-task mapping.

Fields:

* id

---

## Donations

Stores donor information.

Fields:

* id

# AI Resource Allocation Module

Create a dedicated table for AI-generated allocation plans.

## AI_Allocations

Fields:

* id
* report_id
* priority_level
* priority_score
* allocated_resources
* assigned_volunteers
* allocation_reason
* status
* created_at

Status:

* Pending
* Approved
* Rejected

---

# Data Flow

Reporter submits report

↓

Report stored in database

↓

NGO opens AI Resource Allocation Engine

↓

System fetches:

* Report data
* Resource inventory
* Volunteer availability

↓

Gemini generates allocation plan

↓

Plan stored in AI_Allocations table

↓

NGO/Admin reviews

↓

Approved allocation updates resources and assignments
